# Intent

Maintain a popular open-source Python library for data validation as a healthy, trusted project. Ongoing responsibilities span code correctness, release engineering, issue triage, PR review, documentation, versioning and deprecations, security advisories, ecosystem compatibility (new Python versions, new type-checker behavior), contributor relations, and long-range roadmap stewardship.

# Criteria

Healthy ongoing project: main branch always green, releases cut on a predictable cadence, open issue count not growing unbounded, PRs reviewed within reasonable SLO, docs accurate for current release, security advisories patched promptly, contributors onboarded and retained, roadmap public and kept up to date. No terminal state — the gap is filled moment-to-moment by the ongoing work of keeping the project in good standing.
